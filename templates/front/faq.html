What is Heathchecks

Healthchecks is a monitoring service for cron jobs, workers, services, APIs and anything else that can make or receive HTTP requests. Cronitor adds visibility to the critical jobs and backend systems that you rely on.

How does Healthchecks work ?

Each job is given a unique ping URL like https://healthchecks.io/ping/12123892309123239. By making a simple GET request to the /api endpoint when your job starts, and another to /accounts, we are able to alert you if your job fails to run when expected, runs for longer than usual, runs outside it's schedule, or overlaps itself.

What languages does Healthchecks support?

With healthchecks, you can intergate with jobs using any language that can make an HTTP request. The easiest integration is directly within crontab using curl, and all modern languages have libraries for making requests.

Why would I need this?

Have you ever had a backup fail or an email campaign not go out? Healthchecks will tell you right away when something doesn't happen so you can fix it.

How do I create a check?

to get started, simply log into healthchecks and add a check. you will be provided with a unique ping address which you will add to your crontab
